Detailed Description

The WindowsConfiguration.RebootOption enumerated type specifies what should be done to the guest after the customization.

This enumeration was added in vSphere API 7.0.0.0.

Member Enumeration Documentation

enum vmware::vcenter::guest::WindowsConfiguration::RebootOption::Values

C# enum whose values can be used in switch statements.

Ordinals have no semantic meaning (your code should not rely on them).

Enumerator:
_UNKNOWN 

Special enumeration constant which is used to represent constants that do not exist in the current enum binding.

For example if a client sends a new constant to an older server, the server code will see this special constant, because its enum binding lacks the new constant.

REBOOT 

Reboot the guest after customization.

This constant was added in vSphere API 7.0.0.0.

NO_REBOOT 

Take no action.

Leave the guest OS running after customization. This option can be used to look at values for debugging purposes. This constant was added in vSphere API 7.0.0.0.

SHUTDOWN 

Shutdown the guest after customization.

This constant was added in vSphere API 7.0.0.0.
