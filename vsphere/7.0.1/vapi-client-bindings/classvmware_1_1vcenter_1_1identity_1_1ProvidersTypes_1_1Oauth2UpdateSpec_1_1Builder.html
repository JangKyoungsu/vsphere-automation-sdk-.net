Builder vmware::vcenter::identity::ProvidersTypes::Oauth2UpdateSpec::Builder::SetAuthQueryParams ( global::System.Collections.Generic.Dictionary< string, global::System.Collections.Generic.List< string >>  authQueryParams) [inline]

key/value pairs that are to be appended to the authEndpoint request.

How to append to authEndpoint request: If the map is not empty, a "?" is added to the endpoint URL, and combination of each k and each string in the v is added with an "&" delimiter. Details: If the value contains only one string, then the key is added with "k=v". If the value is an empty list, then the key is added without a "=v". If the value contains multiple strings, then the key is repeated in the query-string for each string in the value. If the map is empty, deletes all params. This attribute was added in vSphere API 7.0.0.0.

Builder vmware::vcenter::identity::ProvidersTypes::Oauth2UpdateSpec::Builder::SetClaimMap ( global::System.Collections.Generic.Dictionary< string, global::System.Collections.Generic.Dictionary< string, global::System.Collections.Generic.List< string >>>  claimMap) [inline]

The map used to transform an OAuth2 claim to a corresponding claim that vCenter Server understands.

Currently only the key "perms" is supported. The key "perms" is used for mapping the "perms" claim of incoming JWT. The value is another map with an external group as the key and a vCenter Server group as value. This attribute was added in vSphere API 7.0.0.0.
